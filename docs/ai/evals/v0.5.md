# v0.5 eval definition: Plan-first adaptive team

Baseline: `edd2d59` (`0.4.0-dev`). Paid provider calls are excluded from default evals.

## Capability evals

- [ ] A persisted runtime event is readable from a fresh Store before any subscriber receives it.
- [ ] A crash after preparing a checkpoint journal is recovered idempotently without event loss or duplication.
- [ ] Timeout/process interruption keeps the Run and Plan resumable; explicit owner cancellation remains terminal.
- [ ] A bounded task-specific Plan is validated, included in Run context, and cannot rewrite completed history.
- [ ] Review-first mode prevents mutating tools until the owner approves the current Plan revision.
- [ ] A Team Mission derives WorkItems from the approved Plan, runs independent readers concurrently, and keeps one writer.
- [ ] Structured review or verification failures create bounded repair/re-verify work instead of reaching Lead.
- [ ] Refresh and SSE reconnect reconstruct the same Plan revision and current step in the browser.

## Regression evals

- [ ] Provider/model selection remains server-validated and credential-filtered.
- [ ] Codex streaming and completed-tool replay remain compatible.
- [ ] Verifier evidence is still mandatory before Lead completion.
- [ ] No journal, Plan, event, Handoff, log, or UI payload stores secrets, private reasoning, full prompts, raw tool arguments, or unbounded output.
- [ ] Workspace containment, one-writer policy, budgets, cancellation, and completion verification remain fail-closed.
- [ ] Schema v1-v4 migration remains explicit and unknown versions fail closed.

## Deterministic graders

- Go unit/component tests for state machines, journal crash stages, migration, adaptive graph construction, repair bounds, and security filters.
- HTTP/SSE integration tests for ordering, replay, approval conflicts, cancellation, and resume.
- Browser E2E for Plan review, live progress, refresh, disconnect, failure, and recovery.
- `gofmt`, normal/race tests, vet, native/cross builds, Compose validation, and Docker acceptance.

## Success metrics

- Critical persistence/recovery regression scenarios: `pass^3 = 100%`.
- All deterministic capability and regression evals pass at `pass@1` before PR review.
- No skipped default test; live Codex smoke remains explicit opt-in and reports separately.
