# v0.5 eval definition: Plan-first adaptive team

Baseline: `edd2d59` (`0.4.0-dev`). Paid provider calls are excluded from default evals.

Fixture-driven graders live in `internal/evals` (fixtures under `internal/evals/testdata/`) plus one `eval_test.go` in each package whose unexported hooks they exercise (`internal/team`, `internal/session`, `internal/web`). All fixtures decode with `DisallowUnknownFields` and run as ordinary Go tests, so they execute in CI with `go test ./...`.

## Capability evals

- [x] A persisted runtime event is readable from a fresh Store before any subscriber receives it. — `TestCommitEventIsDurableBeforeReturn` (session), `TestCommitAndPublishMakesEventDurableBeforeSubscriberDelivery` (web), `TestRecoveryFixtures` (session)
- [x] A crash after preparing a checkpoint journal is recovered idempotently without event loss or duplication. — `TestRecoveryFixtures` (session; all three journal stages × repeated recoveries), `TestLoadRecoversPreparedCommitJournalExactlyOnce` (session)
- [x] Timeout/process interruption keeps the Run and Plan resumable; explicit owner cancellation remains terminal. — `TestTeamCancellationAndTimeoutHaveDistinctRecoverySemantics` (web), `TestCancelIsTerminalAndInterruptIsResumable` / `TestInterruptAndCancelHaveDistinctRecoverySemantics` (session), `TestExplicitCancellationIsTerminal` (agent)
- [x] A bounded task-specific Plan is validated, included in Run context, and cannot rewrite completed history. — `TestPlanFidelityFixtures` (evals; per-op `taskplan.Validate`, monotonic revisions, rejected ops preserve state), `TestPlanCancellationKeepsCompletedHistory`, `TestValidateRejectsUnknownAndLogicallyCompleteActivePlan` (taskplan)
- [x] Review-first mode prevents mutating tools until the owner approves the current Plan revision. — `TestReviewPlanWaitsForApprovalBeforeTeamExecution` (web), Playwright `review-plan.spec.ts`
- [x] A Team Mission derives WorkItems from the approved Plan, runs independent readers concurrently, and keeps one writer. — `TestLaunchTeamRunCreatesAndStreamsDurablePlan` (web), `TestCoordinatorParallelReadersAndSingleWriter`, `TestMissionDependencyAndWriterRules` (team)
- [x] Structured review or verification failures create bounded repair/re-verify work instead of reaching Lead. — `TestTeamVerificationFixtures` (team; fail-closed without verifier evidence, bounded requeue, attempts exhausted, severity-gated repair skip via `advisory-findings-skip-repair` and `blocking-findings-run-repair`), `TestCoordinatorSkipsRepairAfterAdvisoryReview`, `TestCoordinatorRunsRepairAfterBlockingReview`, `TestCoordinatorBlocksLeadWithoutVerifierEvidence`, `TestVerifierCannotCompleteWithoutPassingChecks`, `TestVerifierFailureMarksLivePlanFailed`
- [x] Refresh and SSE reconnect reconstruct the same Plan revision and current step in the browser. — `TestPersistentSessionAPIAndEventReplay` (web), Playwright `review-plan.spec.ts`

## Regression evals

- [x] Provider/model selection remains server-validated and credential-filtered. — `TestLoadDefaultsAndRejectUnknown`, `TestProviderPresetsFillOnlyMissingValues`, `TestPlanModeDefaultsAndRejectsUnknownValues`, `TestAPIKeySettingsControlAvailableCatalogWithoutLeakingSecret` (config/web)
- [x] Codex streaming and completed-tool replay remain compatible. — `TestResponsesStreaming`, `TestResponsesStreamingBuildsToolCallFromOutputItemDone`, `TestReasoningContentIsEncryptedAtRestAndReplayed`, `TestGenerateStreamingToolCall` (model)
- [x] Verifier evidence is still mandatory before Lead completion. — `TestTeamVerificationFixtures` (team), `TestCoordinatorBlocksLeadWithoutVerifierEvidence` (team)
- [x] No journal, Plan, event, Handoff, log, or UI payload stores secrets, private reasoning, full prompts, raw tool arguments, or unbounded output. — `TestOwnerSummaryFixtures` (web; completion event carries no prompt and stays bounded), `TestStoreRejectsSecrets`, `TestRedact`, `TestProjectMemoryPersistsOnlyBoundedRedactedFacts` (storage/session)
- [x] Workspace containment, one-writer policy, budgets, cancellation, and completion verification remain fail-closed. — builtin workspace/policy tests, `TestMissionDependencyAndWriterRules`, `TestCoordinatorStopsWhenWorkerResultExceedsBudget` (team)
- [x] Schema v1-v4 migration remains explicit and unknown versions fail closed. — `TestSchemaV1MigrationAndVisibleHistory`, `TestSchemaV2MigrationKeepsSingleAgentSession`, `TestSchemaV3MigrationAddsOptionalRunPlans`, `TestSchemaVersionAndCorruptCheckpoint` (session)

## Deterministic graders

- Go unit/component tests for state machines, journal crash stages, migration, adaptive graph construction, repair bounds, and security filters.
- Checked-in fixture graders: `TestPlanFidelityFixtures` and `TestHandoffQualityFixtures` (`internal/evals`), `TestTeamVerificationFixtures` (`internal/team`), `TestRecoveryFixtures` (`internal/session`), `TestOwnerSummaryFixtures` (`internal/web`).
- HTTP/SSE integration tests for ordering, replay, approval conflicts, cancellation, and resume.
- Browser E2E for Plan review, live progress, refresh, disconnect, failure, and recovery.
- `gofmt`, normal/race tests, vet, native/cross builds, Compose validation, and Docker acceptance.

## Success metrics

- Critical persistence/recovery regression scenarios: `pass^3 = 100%`.
- All deterministic capability and regression evals pass at `pass@1` before PR review.
- No skipped default test; live Codex smoke is explicit opt-in via `GOHERMIT_LIVE_CODEX_SMOKE=1 go test ./internal/evals/ -run TestLiveCodexSmoke` or the manual `live-smoke` CI job (`workflow_dispatch` only), and reports separately. Paid calls stay disabled by default.
